Deals don't die at the demo — they die in the ninety days between "we choose you" and a signature, in security reviews nobody scheduled and legal queues nobody owned. The mutual close plan is a shared, dated, named-owner timeline from today to go-live, built backwards from the prospect's own deadline. It operationalises [The Mutual Close Plan](docs/07-commercial-influence/navigating-procurement-legal.md?id=_744-the-mutual-close-plan), [Compressing 18-Month Procurement Cycles](docs/02-deal-orchestration/managing-deal-velocity.md?id=_231-compressing-18-month-procurement-cycles), and [Security Reviews, InfoSec Questionnaires & Legal Red-Line](docs/07-commercial-influence/navigating-procurement-legal.md?id=_742-security-reviews-infosec-questionnaires-legal-red-line).

| | |
|---|---|
| **When to use** | The moment a technical win is declared — before commercial negotiation starts |
| **Owner** | AE drives; SE owns the technical/security gates |
| **Time to complete** | ~25 minutes to draft; one call with the champion to make it *mutual* |
| **Output goes to** | Champion + prospect procurement, both executive sponsors; reviewed weekly |

---

## The Template

```markdown
# Mutual Close Plan — [Account] × [Vendor]
**Anchored to:** [prospect's business deadline — fiscal close / renewal date / launch] = [DATE]
**Working backwards, signature must land by: [DATE]**

## The Path (every row has ONE owner per side and a real date)
| # | Gate | Vendor owner | [Account] owner | Target date | Status | Slip impact |
|---|------|--------------|-----------------|-------------|--------|-------------|
| 1 | Technical win confirmed in writing | [SE] | [Eval lead] | | ✅ | — |
| 2 | Executive briefing / decision meeting | [AE] | [Champion] | | | |
| 3 | Security questionnaire submitted | [SE] | [InfoSec contact] | | | +[X] wks if late |
| 4 | Security review passed | [SE] | | | | |
| 5 | Commercial proposal delivered | [AE] | | | | |
| 6 | Legal red-line rounds (assume [2]) | | [Legal contact] | | | +[X] wks/round |
| 7 | Procurement vendor onboarding | | [Procurement] | | | |
| 8 | Signature | [AE] | [Economic buyer] | | | |
| 9 | Kickoff / phase 1 start | [SE→Delivery] | | | | |

## Known Friction — surfaced NOW, not discovered in week 6
- Security review queue time (asked, not assumed): [answer]
- Legal turnaround per round: [answer] · Procurement vendor-setup lead time: [answer]
- Holiday/freeze windows between now and signature: [list]

## Weekly Health Check (every [Friday], 10 min, AE+SE+champion)
- Gates ON TRACK: __ · SLIPPED: __ → escalation sent to both sponsors? [Y/N]
- **Rule: one gate slips twice → executive sponsors are notified with the revised go-live date.**

## The Sentence That Makes It Mutual
> "You told us [go-live] by [date] matters because [their reason]. This plan is the only
> path that gets there — where do you see risk in it?"
```

---

## How to Fill It In

1. **Anchor to their date, never yours.** A close plan built around your quarter-end is sales pressure wearing a spreadsheet. Built backwards from *their* fiscal event or launch, every slip becomes a threat to their own commitment — and their team starts pushing procurement for you.
2. **Ask the queue-time questions out loud.** "How long does your security review typically take?" asked in week one is a planning input; discovered in week nine, it's a lost quarter. The Known Friction section is where deals stop dying of surprise.
3. **One name per gate, both sides.** "Their legal team" is not an owner. A close plan where prospect-side boxes say department names instead of people is a wish, and everyone involved knows it.
4. **The champion must co-author it.** Draft it, then hand your champion the marker — every edit they make converts the document from your timeline into their commitment. A close plan the prospect never touched is just your forecast with extra formatting.

> **Failure Mode:** The close plan written, sent, and never mentioned again. By week four, three gates have silently slipped, nobody re-dated the plan, and it now proves the opposite of what it was built for — that neither side is managing the close. The weekly ten-minute health check *is* the artifact; the table is just what makes it fast.

🔵 **Senior** — The technical win gets you chosen. The mutual close plan is what turns chosen into signed before the momentum decays.
